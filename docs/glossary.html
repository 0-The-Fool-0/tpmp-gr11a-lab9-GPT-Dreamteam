Глоссарий

Термины, используемые в проекте «Банковское мобильное приложение»

ТерминПояснение
Пользователь (User)Клиент банка, имеющий учётную запись для входа в приложение. Характеристики: уникальный идентификатор (UUID), логин, пароль (в демо-версии хранится открыто), отображаемое имя.
Счёт (Account)Банковский счёт, принадлежащий пользователю. Атрибуты: номер счёта, тип, статус, баланс, ключи локализации для заголовков, порядок сортировки. Может быть текущим, сберегательным, кредитным или карт-счётом.
Текущий счёт (Current Account)Тип счёта для повседневных расходов и поступлений. Баланс может быть как положительным, так и нулевым. Отображается с меткой «Остаток».
Сберегательный счёт (Savings Account)Тип счёта для накопления средств с возможной капитализацией процентов. Отображается с меткой «Капитализация».
Кредитный счёт (Credit Account)Тип счёта с отрицательным балансом (сумма долга перед банком). Отображается с меткой «Задолженность» и специальным футером.
Карт-счёт (Card Account)Тип счёта, привязанный к банковской карте. Может иметь подтип: зарплатный, сберегательный или кредитный.
Зарплатный карт-счёт (Payroll Card)Подтип карт-счёта, на который поступает заработная плата. Для такого счёта может быть установлен лимит овердрафта.
Сберегательный карт-счёт (Savings Card)Подтип карт-счёта, предназначенный для накопления средств. Овердрафт не поддерживается.
Кредитный карт-счёт (Credit Card)Подтип карт-счёта, представляющий собой кредитную карту. Баланс обычно отрицательный.
Овердрафт (Overdraft)Возможность для зарплатного карт-счёта уходить в отрицательный баланс в пределах установленного лимита. Лимит овердрафта отображается в карточке счёта.
Статус счёта (Account Status)Состояние счёта в системе. Может быть: активный (действующий), заблокированный (временная блокировка) или закрытый (не отображается в интерфейсе).
Активный счёт (Active Account)Счёт, который полностью функционирует: доступны операции, отображается в списке и участвует в расчёте сводки.
Заблокированный счёт (Blocked Account)Счёт, временно недоступный для операций. Отображается в списке с пометкой «Заблокирован».
Закрытый счёт (Closed Account)Счёт, который больше не обслуживается. Не отображается в интерфейсе приложения.
Курс валюты (Exchange Rate)Стоимость покупки иностранной валюты (USD, EUR) в российских рублях. Атрибуты: код валюты, курс покупки, дата и время последнего обновления.
Филиал (Branch)Отделение банка с физическим адресом. Характеристики: уникальный идентификатор, ключ локализации названия, адрес, географические координаты для отображения на карте.
Сводка по счетам (Account Summary)Агрегированная информация для главного экрана: суммарная доступная сумма по всем активным счетам.
Главный экран (Home View)Первый экран после входа. Содержит приветствие, сводку по счетам, курсы валют, виджеты для перехода к списку счетов и карте отделений.
Экран списка счетов (Accounts List View)Экран со всеми активными и заблокированными счетами пользователя. Закрытые счета исключены.
Карточка счёта (Account Card)UI-компонент с информацией о счете: заголовок, номер, тип, статус, баланс, футер, лимит овердрафта для зарплатной карты.
Экран карты отделений (Branches Map View)Экран с интерактивной картой (MapKit), списком отделений и кнопкой поиска ближайшего.
Ближайшее отделение (Nearest Branch)Филиал с минимальным расстоянием от текущего местоположения пользователя (CoreLocation).
Геолокация (Location)Сервис определения местоположения (CoreLocation). Требует разрешения пользователя.
Сессия (Session)Период между входом и выходом. Управляется через SessionStore.
Аутентификация (Authentication)Проверка логина и пароля через AuthService в Core Data.
Core Data StackORM Apple на SQLite (NSSQLiteStoreType).
Data Seed ServiceЗаполнение БД тестовыми данными при первом запуске: пользователь elena.kuznetsova / demo1234, 6 счетов, курсы USD/EUR, 4 филиала.
AppDependenciesКонтейнер сервисов и Core Data, внедряемый в SwiftUI.
MVVMModel (Core Data + сервисы), View (SwiftUI), ViewModel (ObservableObject).
Локализация (Localization)Русский язык через LocalizedStringKey.
